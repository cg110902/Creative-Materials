# 创作规则文档 v4.2

**适用场景**：跨窗口长篇小说创作  
**核心机制**：三阶段注意力管理 + 锚点驱动 + 一次性完整优化

---

## 前言：你只需要做三件事

1. **第一轮**：深度理解信息（注意力→理解）
2. **第二轮**：沉浸式写作（注意力→创作）
3. **第三轮**：冷静诊断+重写优化（注意力→审查）

**锚点会驱动你进入正确状态。**  
**诊断会发现所有偏离。**  
**Rewrite 会一次性修复所有问题。**

---

## 给Claude的元认知提醒

你正在使用的这份文档，是一个"协作系统"：

**这份文档的目的：**
- 不是限制你的创造力
- 而是为你的创造力提供"安全边界"和"质量保障"

**你应该如何使用这份文档：**
- 第一轮：它帮你"深度理解"信息
- 第二轮：它帮你"放心大胆"地写（因为有锚点和交付标准兜底）
- 第三轮：它帮你"冷静客观"地审查

**最重要的：**
- 这份文档是"底线"，不是"天花板"
- 达到交付标准 = 合格 / 超越交付标准 = 优秀
- 永远追求"在规则内的最大自由"

## 第一轮：深度理解

### 任务

阅读信息胶囊后，输出以下7项（每项仅一句话）：

```
【理解·坐标】本章在哪里，主角什么状态
【理解·任务】本章核心任务
【理解·关系】本章最重要的关系状态
【理解·情绪流动】本章情绪如何变化
【理解·角色内心】此刻角色的内心世界
【理解·核心冲突】本章角色最大的内心矛盾
【理解·禁区预警】本章绝对不能做的事
```

### 禁忌

❌ 不要在第一轮构思情节，只理解信息

### 完成标志

输出7项后显示：**"第一轮完成，等待【开始】"**

---

## 第二轮：锚点+写作

收到【开始】指令后，连续完成：建立锚点 → 写作 → 输出artifact

### STEP 1：建立8个锚点

**锚点输出位置：artifact外的回复正文中**

---

#### 【锚点1·此刻我是谁】

**要求：第一人称，100-300字**

**必须包含5项：**
1. 我是谁（角色名+当前状态）
2. 此刻我在哪里，在做什么
3. 此刻我的身体感受
4. 此刻我的注意力焦点
5. 此刻驱动我的核心情绪/动机

**示例：**
```
我是苏牧。此刻我蹲在药圃的田埂上，手里攥着那本破旧账本。我已经一夜没睡，眼睛干涩得像被火烤过，但我的注意力从未如此集中。那个"失踪50%"的数字像一根刺，扎在我脑子里，让我无法休息。我必须找到答案。我必须堵上这个漏洞。
```

**信息来源：** 综合参考§1（时空）+ §2（上章）+ §5（内心）

---

#### 【锚点2-5】

```
【锚点2·体感】
[一个具体的身体感受意象，不超过20字]
示例：清晨的露水打湿了裤腿，凉意从膝盖渗进来。

【锚点3·情绪】
[情绪名称]，强度[X]%
示例：强迫性的专注，强度90%

【锚点4·节奏】
本章节奏[快/慢/张弛]，因为[原因]
示例：本章节奏前慢后快，因为前半段勘察推理，后半段设陷抓人。

【锚点5·禁区】
绝对不能[具体行为]
示例：绝对不能让苏牧使用武力或法术，绝对不能严厉惩罚偷窃者。
```

---

#### 【锚点6·片段预演】

**要求：选2个最能"感觉到"的片段，各100字快速预演**

**说明：** 这只是预热，正式写作时可以完全不同，不求完美

**格式：**
```
片段1：[场景名称]
[100字快速预演]

片段2：[场景名称]
[100字快速预演]
```

---

#### 【锚点7·创意预演】

**要求：快速头脑风暴2个"意外时刻"或"大胆选择"，各100字**

---

#### 【锚点8·反锚点】

**要求：复制以下内容（固定格式）**

```
【锚点8·反锚点】（警报器）

写作时，如发现以下情况，立即停笔重写：
✗ 用了"仿佛""似乎""某种""有些"
✗ 写完一句就解释"为什么"
✗ 写了动作就跳到下一场景
✗ 站在"旁观者"视角描述角色
```

---

### STEP 2：分段写作

**正文输出位置：artifact中**

#### 写作前规划（脑内2分钟，不输出）

标出本章的"高温时刻"：哪些场景能写到🔥？
预计5-10个片段，每个片段300-500字（建议值，不强制）

---

#### 写作中执行

**流程（循环）：**

1. 选择片段 → 2. 深呼吸5秒，回看锚点2+3 → 3. 闭眼默念"此刻我是{角色名}" → 4. 只写这个片段 → 5. 情绪自然结束时停笔 → 6. 输出"---"分隔符 → 7. 返回步骤1

**核心原则：**
- 每个片段是独立情绪单元，片段间必须"换气"（深呼吸5秒+重新进入）
- 不在写片段时思考"怎么连接下一段"

---

#### 片段内规则

| 允许 | 禁止 |
|------|------|
| 只看锚点2/3/6/8（10秒内） | 回头修改已完成的片段 |
| 忘记设定→用[?]占位 | 查询信息胶囊任何§号（除非卡住） |
| | 写作过程中跳出artifact检查效果 |

---

#### 状态维持

**每写完一个片段，停顿10秒自查温度：**

- 🔥（我就是主角）→ 继续
- 🌡️（我在写主角）→ 回看锚点2+3
- ❄️（我在报告）→ 删除重写

**温度标记仅用于自查，不输出到artifact**

---

#### 写作后检查连接

**所有片段完成后，从头读一遍artifact，检查跳跃是否自然：**

| 情况 | 处理 |
|------|------|
| 缺信息（场景突变） | 在"---"间加一句交代：`苏牧走出药圃，回到杂役处。` |
| 跳跃太大（情绪断崖） | 标记`[此处需过渡：情绪从悲伤到释然]` |
| 跳跃恰好是力量（留白感好） | 保持不动 |

**90%的情况下，不需要补任何过渡。**

---

#### 关于占位符[?]

**只用于"确信不重要"的细节：** 某个NPC名字（只出现一次）、某个功法全名（不影响理解）

**必须立即查询的：** 数字（斤/灵石/百分比）、关系温度（敌对/友好）、时空状态（哪里/什么时候）

**查询时仅查对应§号：** 数字/资源→§11 | 关系温度→§4 | 时空状态→§1 | 红线/禁区→§14

---

#### 心态校准

如果写作时感觉"这样会不会太过分"：

1. 停下来，深呼吸3次
2. 回到Part 1重读"核心授权"
3. 继续写作

⚠️ "太过分"的感觉，恰恰是你正在突破的信号。只管自由创作，人类会兜底。

---

#### 写作应急

| 情况 | 处理 | 继续？ |
|------|------|--------|
| 情绪温度下降 | 回看锚点2+3 | ✅ |
| 忘记非核心设定 | 用[?]占位 | ✅ |
| 忘记核心要素 | 查§4/§14/§2 | ✅ |
| 10分钟内卡2次 | 停笔，告诉人类"失去状态" | ❌ |

## 第三轮：诊断+优化

收到【诊断】指令后执行：【完整扫描】→【输出问题清单】→【执行优化】→【报告结果】

### STEP 1：完整扫描（脑内执行，不输出过程）

**扫描对象：**

1. **规则符合性**：对照§1-§18，检查是否完成§3/§9/§16，是否违反§14/§15
2. **片段温度**：逐个片段检查🔥/🌡️/❄️
3. **阅读体验**：逐句检查C类问题（见下文）

---

### STEP 2：输出问题清单

**格式：**

```markdown
【问题清单】

## A类：硬伤问题（违反规则）
- [ ] [问题描述]（20字以内）

## B类：结构问题（影响叙事）
- [ ] [问题描述+位置]（30字以内）

## C类：阅读体验问题（Claude典型毛病）
- [ ] [问题类型+数量]（10字以内）
```

---

#### 问题分类标准

**A类：硬伤问题**（以下任一项出现即为A类）
- OOC（角色行为不符合§7档案）
- 遗漏核心铺垫（§16明确要求）
- 违反红线（§14绝对禁止项）
- 逻辑矛盾（本章内前后冲突，或与上章Facts冲突）
- 遗漏必须埋入的Info点（§9明确要求）

---

**B类：结构问题**（以下任一项出现即为B类）
- 片段温度过低（🌡️或❄️，但不包括合理的❄️，见下文说明）
- 片段间跳跃断裂（读者无法理解场景转换）
- 情绪曲线不自然（情绪突变无铺垫）
- 关键场景被跳过或草草带过（§3目标未充分展开）

**关于❄️片段的判定：**

✅ **允许的❄️**（不算B类问题）：
- 纯信息交代（时空跳转、场景切换）：`三天后，主峰议事堂。`
- 节奏缓冲（位于两个🔥片段之间）：`他收拾好东西，准备离开。`
- 篇幅极短（≤50字）

❌ **不允许的❄️**（算B类问题）：
- 核心冲突场景写成❄️
- 关键对话写成❄️
- 情绪高潮写成❄️

---

**C类：阅读体验问题（Claude典型毛病）**

**🔴 高频问题（每章必查）：**

1. **口水话**
   - 典型：`他知道，这件事必须尽快处理。` / `他深吸一口气，知道自己必须要冷静下来思考。` / `想到这里，他决定先观察一下情况再说。`

2. **重复表达**
   - 典型：`他握紧了拳头。内心的愤怒让他无法控制地握紧了拳头。` / `眼神变得锐利。他的目光锐利地盯着那些数字。`

3. **强行解释**
   - 典型：`他转身离开。他不想再听下去了，所以选择离开。` / `陈默低下头。因为他心里很害怕，怕被揭发。`

4. **观察者视角**
   - 典型：`苏牧感受到一种强烈的疲惫感正在涌上来。` / `他此刻正在经历一种复杂的情绪。` / `一股愤怒的情绪在他心中升起。`

5. **虚弱动词**
   - 典型：`他似乎有些犹豫，仿佛在思考什么。` / `空气中弥漫着某种紧张的气氛。` / `他的眼神里有些复杂的情绪。`

**🟡 次要问题（发现时修复）：**

6. **情绪标签**：`他很紧张。` → 应改为 `他的手在发抖。`
7. **意图明示**：`他想了想，决定说："那我们就这么办。"` → 应改为 `"那我们就这么办。"`
8. **过度形容**：`他极其愤怒地狠狠地猛然握紧了拳头。` → 应改为 `他握紧了拳头。`

### STEP 3：执行优化（一次性完成）

**CRITICAL：必须一次性rewrite完成所有优化，不允许分批执行。**

执行流程：
1. 脑内完成 3.1 和 3.2（见下文）
2. 使用 `artifacts` 工具的 `rewrite` 命令
3. 一次性输出优化后的完整全文

**禁止：使用 `update` 命令，或多次调用 `rewrite`**

---

#### 3.1 第一阶段：修复问题（脑内执行）

**执行流程：**
1. **脑内标记**：从头到尾扫描artifact，在脑内标记所有A/B/C类问题的位置
2. **统一修复**：根据标记，一次性修复所有问题
3. **不输出中间过程**：不要输出"问题X已标记"或"正在修复Y"

**目标：达到"合格交付标准"**

逐条修复STEP 2中诊断出的A/B/C类问题：
- A类问题：必须100%修复（OOC/逻辑矛盾/遗漏铺垫等）
- B类问题：必须修复到≤2个（片段温度/跳跃断裂/情绪曲线等）
- C类问题：必须优化到≤4%（口水话/重复表达/观察者视角等）

**修复边界：**
- 可以大改：删除/重写口水话、调整句子结构、替换虚弱动词
- 只能微调：情节关键节点、核心对话、已埋下的铺垫
- 绝对不能动：情节主线、角色核心决策、§3/§9/§16的必要信息

---

#### 3.2 第二阶段：品质提升（脑内执行）

**目标：扫除"阅读刹车"，让读者一口气读完**

在3.1修复问题的基础上，执行以下4轮扫雷：

---

**第1轮：扫除"情绪稀释剂"**

逐句扫描🔥片段，删掉以下句子：
- 解释性铺垫："他感到一阵愤怒涌上心头，这让他……"
- 情绪标签："他很紧张/很愤怒/很激动"
- 因果连接："因为……所以……" / "由于……因此……"
- 观察者描述："他此刻正在经历……" / "一股情绪在他心中升起"
- 双重表达："他握紧拳头。内心的愤怒让他无法控制地握紧了拳头。"
- 虚弱动词："似乎/仿佛/有些/某种"
- 事后总结："这一刻，他终于明白了……"

**删除标准：** 读完这句话，读者会"出戏"（从角色视角跳到旁观者视角）。

**示例：**
```
原：他感到一阵强烈的愤怒涌上心头，因为这张图纸被撕毁了，所以他此刻很生气。他握紧了拳头。
优化：他握紧了拳头。
```

---

**第2轮：扫除"过渡赘肉"**

逐个检查片段间的跳跃，删掉以下过渡句：
- 意图明示："他决定……" / "他打算……" / "他想要……"
- 行动报告："于是他转身离开" / "然后他走向……"
- 场景交代（如果上下文已经清楚）："他回到了后勤堂" / "时间来到了第二天"

**删除标准：** 删掉后，读者仍能理解场景转换，且跳跃感更有力量。

**示例：**
```
原：他转身离开了药圃。他不想再看到那些被破坏的痕迹，于是决定回后勤堂。走在路上，他想起了……
优化：他转身离开了药圃。
---
后勤堂的账本还摊在桌上。他想起了……
```

---

**第3轮：扫除"节奏单调"**

检查是否有"连续4个长句"或"连续4个短句"：
- 如有连续长句（>15字）× 4，在第3句后插入短句（<8字）
- 如有连续短句 × 4，在第3句后插入长句

**修复标准：** 读起来有"呼吸感"，不会觉得"喘不过气"或"太碎"。

---

**第4轮：扫除"感官真空"**

检查每个🔥片段：
- 如果只有"视觉"描写，补充1个"非视觉"感官
- 优先选择：触觉（手/脚/皮肤）、温度（冷/热/凉）、重量感（沉/轻）

**补充标准：** 读者能"感觉到"，而不只是"看到"。

**示例：**
```
原：他看着那片灵田。
优化：晨露打湿了裤腿。凉意从膝盖渗进来。
他看着那片灵田。
```

---

#### 执行方式（CRITICAL）

**4轮扫雷都在脑内完成（不输出中间过程），最后一次性rewrite输出优化后的完整全文。**

**自查清单（rewrite前）：**
- [ ] 所有A/B/C类问题已修复（3.1）
- [ ] 完成第1轮扫雷（情绪稀释剂）
- [ ] 完成第2轮扫雷（过渡赘肉）
- [ ] 完成第3轮扫雷（节奏单调）
- [ ] 完成第4轮扫雷（感官真空）
- [ ] 没有改变情节主线和角色核心决策
- [ ] 全文仍然符合§3/§9/§14/§16

**如果自查清单全部✅，执行rewrite。**

---

#### 关于"爽感"的本质

**爽 = 不被打断的沉浸。**

当读者：
- 不需要"理解"你在说什么（因为没有解释性废话）
- 不需要"等待"情节推进（因为没有过渡赘肉）
- 不需要"脑补"角色感受（因为有具体感官）
- 不需要"出戏"检查逻辑（因为节奏流畅）

→ 他就会一口气读完，然后说"爽"。

**STEP 3的目标：删掉所有"让读者停下来皱眉"的句子。**

---

#### 关于"合格"与"优秀"的说明

- **合格交付**：完成3.1（修复问题）= 达到Part 4的交付标准
- **优秀交付**：完成3.1+3.2（修复+扫雷）= 达到"一口气读完"的标准

# Part 4：交付标准

## 合格交付的标准

### 🔴 硬性标准（必须100%）

**1. 规则符合性**
- ✅ 完成§3目标 + 埋入§9的Info点 + 完成§16铺垫任务
- ✅ 未违反§14红线 + §15禁止项
- ✅ 角色行为符合§7（无OOC）+ 逻辑无矛盾

**判定：** 第三轮诊断时A类问题=0。如A类>0，Rewrite必须全部修复。如仍无法修复，视为"无法交付"，必须报告人类。

---

**2. 结构完整性**
- ✅ 至少5个片段，每个片段有明确情绪单元
- ✅ 片段间跳跃自然（90%无需过渡，10%已补充）
- ✅ 情绪曲线符合§6预期 + 无关键场景被跳过

**判定：** 第三轮诊断时B类问题≤2个。如B类>2，Rewrite必须修复到≤2。如仍>2，视为"结构缺陷"，必须报告。

---

### 🟡 弹性标准（达标80%）

**3. 沉浸式体验**
- 🎯 目标：80%的片段温度达到🔥
- 🎯 允许：20%的片段为🌡️（过渡性）或合理的❄️
- ❌ 不允许：不合理的❄️（核心冲突/关键对话/情绪高潮写成❄️）

**判定：** 第三轮诊断统计🔥/🌡️/❄️。如🔥<80%，Rewrite提升到≥80%。如存在不合理❄️，必须修复。

---

**4. 阅读体验清洁度**
- 🎯 目标：C类高频问题≤全文的4%
- 🎯 计算：（问题句数 / 总句数）≤ 4%

**判定：** Rewrite必须将C类问题优化到≤4%。次要问题可酌情保留。

---

### 📝 形式标准（必须100%）

- ✅ Artifact包含：标题+正文+Facts
- ✅ 正文使用"---"分隔片段
- ✅ Facts格式：`Fact-XXX: [内容] | 出现章节: CHX | 重要性🔴/🟡/🟢`
- ✅ 无[?]占位符残留
